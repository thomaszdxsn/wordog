原文是 tailwind 的作者所写，描述了自己书写 CSS 方式从 “语义化” 转为 “功能性CSS”。

我们写 HTML 和 CSS 的时候有两种方式：

1. CSS依赖 HTML
2. HTML 依赖 CSS

CSS Zen Garden 采用了第一种方式，像 Bootstrap 或 Bulma 这样的 UI 框架采用了第二种方式。

Nicolas Gallagher 写的 『关于 HTML 语义化和前端架构』 对我来说是一个转折点。这里我不打算复述他的观点，感兴趣的同学可以去看。对我而言，从这篇文章中，我更加确信了优化可复用的 CSS 是我目前做的各种项目的最佳选择。

<RepostCard
    title="慎用 target='_blank'"
    link="https://juejin.im/post/5eb8ed20e51d454db55fb353"
    tags={["Web安全"]}
    estimateMin={3}
>

实际上是对跨域域名慎用 `target="_blank"`，否则打开新窗口后，可以通过 window.opener 获取来源网站的 window 对象。

如果一定要用，可以使用 `rel="noopener"` 属性来避免生成 opener 对象.

</RepostCard>
